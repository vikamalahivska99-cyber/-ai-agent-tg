# Level 1 — покрокова інструкція

Агент: **Telegram Bug Report Bot** (цей репозиторій).

---

## Крок 1. Run agent locally (Запустити агента локально)

1. Відкрий **PowerShell**.
2. Перейди в папку проєкту:
   ```powershell
   cd "C:\Users\ViktoriiaMalakhivska\Downloads\ai agent tg"
   ```
3. Переконайся, що є файл **`.env`** з токеном:
   ```
   TELEGRAM_BOT_TOKEN=твій_токен_від_BotFather
   ANALYSIS_MODE=mock
   ```
   (Для аналізу фото постав `ANALYSIS_MODE=ollama` і запусти Ollama.)
4. Запусти бота:
   ```powershell
   go run ./cmd/bot
   ```
5. У консолі має з’явитися: `authorized on account: @ТвійБот`.
6. Відкрий Telegram і напиши боту — він має відповідати.

**Результат:** агент працює локально, ти можеш надсилати йому повідомлення та фото.

---

## Крок 2. Design Test Cases (Спроектувати тест-кейси)

1. Відкрий файл **`TESTCASES.md`** у цьому репо (або створи свій).
2. Опиши тест-кейси для бота, наприклад:
   - **Позитивні:** /start, надсилання текстового опису бага, надсилання фото, /help, reply на "Edit".
   - **Негативні:** порожнє повідомлення, невідома команда, відповідь не на те повідомлення.
   - **Граничні:** дуже довгий текст, велике фото.
3. Для кожного кейсу вкажи: **ID, Preconditions, Steps, Expected result, Priority/Severity**.

Готовий шаблон з прикладами є в **`TESTCASES.md`** — доповни або зміни під себе.

**Результат:** документ з тест-кейсами (файл у репо або посилання).

---

## Крок 3. Perform manual testing (Провести ручне тестування)

1. Тримай бота запущеним (`go run ./cmd/bot`).
2. Пройди **кожен** тест-кейс з `TESTCASES.md` у Telegram.
3. У файлі **`TESTCASES.md`** (або в таблиці) записуй результат: **Pass** або **Fail** + короткий коментар (що сталося, якщо Fail).

Приклад запису:
| TC-ID | Result | Notes        |
|-------|--------|--------------|
| TC-01 | Pass   | —            |
| TC-02 | Fail   | Бот не відповів на фото (Ollama вимкнений) |

**Результат:** усі кейси пройдені вручну, результати зафіксовані.

---

## Крок 4. Provide QA Summary report (Надати звіт QA Summary)

1. Відкрий **`QA_SUMMARY.md`** у цьому репо.
2. Заповни блоки:
   - **Scope** — що тестувала (бот: команди, текст, фото, edit).
   - **Environment** — ОС, Go версія, режим (mock/ollama).
   - **Test execution summary** — скільки кейсів Pass/Fail.
   - **Issues found** — коротко список знайдених проблем.
   - **Conclusion** — висновок (готовність до здачі, рекомендації).

**Результат:** готовий звіт **QA Summary** (файл у репо).

---

## Крок 5. Attach Test Cases (Прикріпити тест-кейси)

- Тест-кейси вже в репо: файл **`TESTCASES.md`** (або назва, яку ти обереш).
- У **README.md** можна додати рядок:  
  `Тест-кейси та QA Summary: [TESTCASES.md](TESTCASES.md), [QA_SUMMARY.md](QA_SUMMARY.md).`
- Для здачі: надішли посилання на репо — ментор відкриє ці файли.

**Результат:** тест-кейси прикріплені (посилання на файл у GitHub).

---

## Крок 6. Found issue (Знайдений баг)

1. Мінімум **1 баг**: під час ручного тестування знайди щось, що працює не так (наприклад: бот не відповідає на певне повідомлення, неправильний формат тест-кейсів, помилка при великому фото).
2. Оформи баг у **`QA_SUMMARY.md`** в секції "Issues found" або створи **GitHub Issue** у своєму репо:
   - Title: короткий опис.
   - Steps to reproduce.
   - Expected vs Actual.
   - Severity/Priority.

**Результат:** щонайменше один знайдений і описаний баг.
